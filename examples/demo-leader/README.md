# 示范产品总监

> 一个脱敏后的领导画像 bundle。它的作用不是影射某个真实人物，而是展示 `leader-skill` 最终生成物应该长什么样。

| 维度 | 结论 |

| 甩锅风险 | 68 |

| 今晚加班概率 | 74 |

## 这个示例为什么值得看

- 它把“这个你先看一下”“这个不复杂吧”“我们再对齐一下”这类高频话术翻译成了行动建议。
- 它展示了一个完整 bundle 的结构，而不是只给一段 README 文案。
- 它保留了 `versions/` 目录，让你能看见版本化输出的形态。

## 建议打开顺序

1. `persona.md`
2. `intent-map.md`
3. `playbook.md`
4. `versions/20260401-initial/`

## 最值得先记住的 3 个动作

- 先给可评审版本，再把范围、优先级和验收标准补成书面。
- 同步前准备一句结论版口径，避免在会上被动改口。
- 每次临时任务都留下一句书面确认，让责任边界和时间边界更清楚。
